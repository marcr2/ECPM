<task type="auto">
  <name>Task 1: Create Celery training pipeline tasks with Redis progress streaming</name>
  <files>backend/ecpm/tasks/training_tasks.py, backend/ecpm/tasks/__init__.py</files>
  <action>
Create `backend/ecpm/tasks/training_tasks.py` implementing the model training pipeline as a Celery chain.

1. Helper function `_publish_progress(task, step: str, status: str, detail: str = "", total_steps: int = 6)` -- publishes JSON to Redis pubsub channel `ecpm:training:progress`. Message format: `{"task_id": task.request.id, "step": step, "status": status, "detail": detail, "total_steps": total_steps, "timestamp": isoformat}`. Uses synchronous redis client (not async) since Celery tasks run in worker processes.

2. Helper function `_get_indicator_data() -> pd.DataFrame` -- uses synchronous SQLAlchemy session (not async -- Celery workers are sync) to query all 8 indicator time series from the observations table, applying the same FRED-to-descriptive-key mapping from computation.py. Aligns to common date range (intersection of all series). Returns DataFrame with columns named by IndicatorSlug values. Fall back: if DB is unavailable, raise with clear error message.

NOTE: Since Celery workers are synchronous, this function must use synchronous SQLAlchemy (create_engine, not create_async_engine). Create a sync engine from settings.database_url (replace +asyncpg with +psycopg2 or use the sync equivalent). If running in test mode (SQLite), handle accordingly.

3. Task `run_stationarity_step(self, data_config: dict) -> dict` -- bind=True, name="ecpm.tasks.training.stationarity". Call _get_indicator_data(), run ensure_stationarity(). Publish progress at start and end. Return {"data": transformed_data.to_json(), "diff_orders": diff_orders, "stationarity_results": per_column_results, "original_last_values": last_row_as_dict}.

4. Task `run_var_step(self, prev: dict) -> dict` -- Reconstruct DataFrame from prev["data"]. Call fit_var_model(). Call generate_forecasts() with diff_orders for inverse differencing. Publish progress. Return {**prev, "forecasts": forecasts_dict, "var_info": {"selected_lag": ..., "aic": ..., "bic": ...}}.

5. Task `run_svar_step(self, prev: dict) -> dict` -- Reconstruct data, refit VAR (needed for SVAR), call build_svar_a_matrix() and fit_svar(). If SVAR fails, log warning but continue (SVAR is supplementary to VAR). Publish progress. Return {**prev, "svar_fitted": bool}.

6. Task `run_regime_step(self, prev: dict) -> dict` -- Reconstruct the crisis index composite from the original (non-differenced) indicator data. Call fit_regime_model(). Publish progress. Return {**prev, "regime": serialized_result}.

7. Task `run_backtest_step(self, prev: dict) -> dict` -- Reconstruct original indicator data (from prev["original_last_values"] or re-fetch). Call run_all_backtests(). Publish progress. Return {**prev, "backtests": list_of_results}.

8. Task `cache_results_step(self, prev: dict) -> dict` -- Write all results to Redis under versioned key prefix `ecpm:model:v{timestamp}`. Keys: `ecpm:model:v{ts}:forecast:{indicator}` for each indicator, `ecpm:model:v{ts}:regime`, `ecpm:model:v{ts}:crisis_index`, `ecpm:model:v{ts}:backtest:{episode_name}`, `ecpm:model:v{ts}:var_info`, `ecpm:model:v{ts}:training_status`. All with TTL 86400 (24h). After all writes, set `ecpm:model:current_version` to the version timestamp (atomic pointer swap). Publish "complete" progress. Return {"version": ts, "status": "complete"}.

9. Function `run_training_pipeline() -> AsyncResult` -- creates the chain: `chain(run_stationarity_step.s({}), run_var_step.s(), run_svar_step.s(), run_regime_step.s(), run_backtest_step.s(), cache_results_step.s())` and returns `.apply_async()`.

Error handling: each task wraps its body in try/except. On failure: publish progress with status="error" and detail=str(exception), then re-raise so Celery marks the chain as failed. The cache_results_step also writes a training_status key with error info if it detects a failure flag in prev.

Update `backend/ecpm/tasks/__init__.py` to import training_tasks so autodiscovery finds the tasks.

  <name>Task 2: Create FastAPI forecasting endpoints with SSE training stream</name>
  <files>backend/ecpm/api/forecasting.py, backend/ecpm/api/router.py</files>
  <action>
1. Create `backend/ecpm/api/forecasting.py` with an APIRouter(prefix="/api/forecasting", tags=["forecasting"]).

Endpoints:

a) `GET /api/forecasting/forecasts?indicator={slug}&horizon={4|8|12|16}` -- Read current_version from Redis (`ecpm:model:current_version`). If no version, return 404 with {"detail": "No model results available. Trigger training first."}. If indicator param given, read `ecpm:model:v{version}:forecast:{indicator}`. If not, read all 8 indicator forecasts. Return ForecastsResponse. Default horizon filter: if requested horizon differs from cached horizon, slice the forecast arrays.

b) `GET /api/forecasting/regime` -- Read `ecpm:model:v{version}:regime` from Redis. Return RegimeResult. 404 if no results.

c) `GET /api/forecasting/crisis-index` -- Read `ecpm:model:v{version}:crisis_index` from Redis. Return CrisisIndex. 404 if no results.

d) `GET /api/forecasting/backtests?episode={name}` -- Read backtest results from Redis. If episode param given, return single BacktestResult. If not, return BacktestsResponse with all episodes. 404 if no results.

e) `POST /api/forecasting/train` -- Import run_training_pipeline from tasks. Call it. Return 202 Accepted with {"task_id": result.id, "status": "started"}. If a training is already running (check Redis key `ecpm:training:active`), return 409 Conflict.

f) `GET /api/forecasting/training/stream` -- SSE endpoint following the exact pattern from status.py fetch_stream. Subscribe to Redis pubsub channel `ecpm:training:progress`. Yield events: "connected", "progress" (with step data), "heartbeat" (every 1s of inactivity). Max timeout 600s (10 min for training). Clean up pubsub on disconnect. Use EventSourceResponse from sse-starlette.

g) `GET /api/forecasting/training/status` -- Read `ecpm:model:v{version}:training_status` from Redis. Return TrainingStatus. If no training has run, return {"status": "idle", "steps": []}.

Redis dependency: use the same _get_redis() pattern from status.py. Accept redis as FastAPI dependency where possible.

Follow existing patterns: structlog logging, Pydantic response_model on endpoints, proper HTTP status codes.
