**financial.py** -- Standalone financial indicator functions (methodology-independent):

`compute_productivity_wage_gap(data: dict[str, pd.Series], window: int = 20) -> pd.Series`:
- Uses FRED series: OPHNFB (output per hour, nonfarm business) and PRS85006092 (real compensation per hour)
- Normalizes both to base 100 at first observation
- Returns: output_index / compensation_index * 100 (values > 100 = gap widening)
- Apply rolling window for smoothing if window > 0

`compute_credit_gdp_gap(data: dict[str, pd.Series]) -> pd.Series`:
- Uses: BOGZ1FL073164003Q (nonfinancial corporate debt) and GDP (nominal GDP)
- Credit-to-GDP ratio = credit / (4-quarter rolling sum of GDP for quarterly data)
- Apply one-sided HP filter with lambda=400,000 (use the recursive approximation from 02-RESEARCH.md)
- Gap = actual ratio - trend
- Return gap in percentage points

`compute_financial_real_ratio(data: dict[str, pd.Series]) -> pd.Series`:
- Uses: BOGZ1FL073164003Q (financial corporate debt as proxy for financial assets) and K1NTOTL1SI000 (tangible/real assets)
- Ratio = financial / real
- Simple division, no filtering

`compute_debt_service_ratio(data: dict[str, pd.Series]) -> pd.Series`:
- Uses: BOGZ1FA096130001Q (interest payments, nonfinancial corporate) and A445RC1Q027SBEA (net operating surplus)
- Ratio = interest / operating_surplus * 100 (as percent)

**computation.py** -- Replace stubs with real implementation:

`async def compute_indicator(slug: str, methodology_slug: str, db: AsyncSession) -> pd.Series`:
1. Get mapper from MethodologyRegistry.get(methodology_slug)
2. Get required series_ids from mapper.get_required_series() + financial series if slug is a financial indicator
3. For each series_id, query Observation table: `SELECT observation_date, value FROM observations WHERE series_id = :sid ORDER BY observation_date`
4. Build `dict[str, pd.Series]` with series_id as key, pd.Series(values, index=dates)
5. Apply LOCF alignment to common frequency (annual for core indicators involving capital stock, quarterly for financial)
6. Call appropriate compute method on mapper based on slug (use a dispatch dict mapping IndicatorSlug -> method name)
7. Return the resulting pd.Series

`async def compute_all_summaries(methodology_slug: str, db: AsyncSession) -> list[dict]`:
1. For each IndicatorSlug, call compute_indicator
2. For each result, extract: latest_value, latest_date, trend (compare last 2 values: "rising"/"falling"/"flat"), sparkline (last 20 values)
3. Return list of summary dicts matching IndicatorSummary schema fields

Add Redis caching: wrap compute_indicator results with cache_get/cache_set using key pattern `indicators:{slug}:{methodology}` with TTL of 3600s (1 hour -- data changes at most daily).
