**api/indicators.py** -- FastAPI router for indicator data:

```python
router = APIRouter(prefix="/api/indicators", tags=["indicators"])
```

`GET /api/indicators/` (overview):
- Query param: `methodology: str = Query("shaikh-tonak")`
- Calls `compute_all_summaries(methodology, db)` from computation module
- Returns `IndicatorOverviewResponse`
- Redis cache with TTL=3600 (indicators change at most when new data is ingested)
- 404 if methodology slug not found

`GET /api/indicators/{slug}` (detail):
- Path param: `slug: str`
- Query params: `methodology: str = "shaikh-tonak"`, `start: Optional[str]`, `end: Optional[str]`
- Calls `compute_indicator(slug, methodology, db)`
- Converts pd.Series to list of IndicatorDataPoint
- Optionally filter by start/end date range
- Returns `IndicatorResponse` with full time-series
- Redis cache with TTL=3600
- 404 if slug unknown or methodology unknown

`GET /api/indicators/{slug}/compare` (comparison):
- Path param: `slug: str`
- For each methodology in `MethodologyRegistry.list_all()`, call `compute_indicator(slug, methodology.slug, db)`
- Return dict mapping methodology slug to IndicatorResponse
- Redis cache with TTL=3600

**IMPORTANT routing note:** The `/api/indicators/methodology` route must be defined BEFORE `/api/indicators/{slug}` in the router to avoid the dynamic route capturing "methodology" as a slug. Use a separate router or careful ordering.

**api/methodology.py** -- FastAPI router for methodology docs:

```python
router = APIRouter(prefix="/api/indicators/methodology", tags=["methodology"])
```

`GET /api/indicators/methodology` (all docs):
- For each mapper in `MethodologyRegistry.list_all()`, call `mapper.get_documentation()`
- Convert to `MethodologyDocResponse` per mapper
- Return list of MethodologyDocResponse
- Redis cache with TTL=86400 (docs change only on code deploy)

`GET /api/indicators/methodology/{methodology_slug}` (single methodology docs):
- Get mapper from `MethodologyRegistry.get(methodology_slug)`
- Return `MethodologyDocResponse`
- Redis cache with TTL=86400
- 404 if slug unknown

**router.py** -- Update to include new routers:
- Import indicator and methodology routers
- Include them in the aggregate router
- Order: methodology router FIRST (more specific prefix), then indicators router

**main.py** -- No changes expected unless routers need explicit mounting beyond router.py.

Follow existing patterns: use `_get_redis()` for Redis access, `structlog.get_logger()` for logging, `Depends(get_db)` for database sessions. Wrap mapper/computation errors with appropriate HTTP exceptions (404 for not found, 500 for computation failures with error detail).
