         - Cache in Redis with 300s TTL (key: series_id + params hash)

         - Helper function `cache_get(key) -> Optional[str]` and `cache_set(key, value, ttl)`

    1. **backend/ecpm/api/status.py** -- Status and fetch endpoints:
       - `router = APIRouter(prefix="/api/data", tags=["status"])`

       - `GET /api/data/status` -- Pipeline status:
         - Query SeriesMetadata for aggregate counts: total, ok, error, stale
         - Find most recent last_fetched across all series
         - Calculate next scheduled run from Celery beat config (settings.fetch_schedule_hour/minute, next occurrence)
         - Collect error details: [{series_id, error} for series with fetch_status="error"]
         - Return FetchStatusResponse

       - `POST /api/data/fetch` -- Manual fetch trigger:
         - Submit `fetch_all_series.delay()` Celery task
         - Return FetchTriggerResponse with task_id
         - Note: Import the task at function call time (not module level) to avoid circular imports if Celery app isn't initialized

       - `GET /api/data/fetch/stream` -- SSE progress streaming (INFR-03):
         - Use `sse_starlette.EventSourceResponse`
         - Subscribe to Redis pubsub channel "fetch_progress"
         - Yield SSE events as they arrive:
           - `event: start` with `{"total_series": N}`
           - `event: progress` with `{"series_id": "...", "status": "ok"/"error", "count": N, "current": N, "total": N}`
           - `event: complete` with `{"status": "done", "summary": {...}}`
         - Timeout after 10 minutes (in case task hangs)
         - Clean disconnect: unsubscribe from pubsub on client disconnect

    2. **backend/ecpm/api/router.py** -- Main router aggregator:
       - Import data.router and status.router
       - `api_router = APIRouter()`
       - Include both sub-routers

       - Import `api_router` from `ecpm.api.router`
       - `app.include_router(api_router)`
       - Keep existing /health endpoint
       - Add startup event to initialize Redis connection pool
