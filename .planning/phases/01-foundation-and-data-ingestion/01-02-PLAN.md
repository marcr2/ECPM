       - Constructor takes `api_key: str`, creates `fredapi.Fred` instance
       - `fetch_series(series_id: str) -> tuple[pd.DataFrame, dict]` method:
         - Wrapped with `@retry` from tenacity: `stop_after_attempt(5)`, `wait_exponential(multiplier=1, min=2, max=60)`, retry on `(ConnectionError, TimeoutError, ValueError)` (ValueError for FRED rate limit 429 errors), `before_sleep` callback that logs retry attempt via structlog
         - fredapi is synchronous -- run `self.fred.get_series()` and `self.fred.get_series_info()` in `asyncio.get_event_loop().run_in_executor(None, ...)` to avoid blocking async callers
         - Return (data DataFrame, info dict with units, frequency, seasonal_adjustment, title, last_updated)
         - Log success with structlog: series_id, observation count
       - `fetch_series_sync(series_id: str) -> tuple[pd.DataFrame, dict]` method:
         - Same as above but synchronous (no executor) for use in Celery workers
         - Same retry decorator
       - Rate limit mitigation: add 0.6s sleep between calls to stay under 120 req/min (per research pitfall #4). Use `time.sleep(0.6)` in sync version.

    3. **backend/ecpm/ingestion/bea_client.py** -- `BEAClient` class:
       - Constructor takes `api_key: str`
       - `fetch_nipa_table(table_name: str, frequency: str = "Q", year: str = "ALL") -> pd.DataFrame` method:
         - Uses `beaapi.get_data(api_key, datasetname='NIPA', TableName=table_name, Frequency=frequency, Year=year)`
         - Wrapped with tenacity retry: same parameters as FredClient
         - Returns DataFrame with columns: LineNumber, LineDescription, TimePeriod, DataValue, etc.
         - Log success with structlog
       - `fetch_fixed_assets_table(table_name: str, year: str = "ALL") -> pd.DataFrame` method:
         - Uses `beaapi.get_data(api_key, datasetname='FixedAssets', TableName=table_name, Year=year)`
         - Same retry wrapper
         - Per research pitfall #6: FixedAssets is a SEPARATE dataset from NIPA
       - `discover_tables(dataset: str = "NIPA") -> pd.DataFrame` method:
         - Uses `beaapi.get_parameter_values(api_key, dataset, 'TableName')`
         - For validating table name codes (per research pitfall #1)

    1. **backend/ecpm/ingestion/pipeline.py** -- `IngestionPipeline` class:
       - Constructor takes `db_session` (sync or async), `fred_client: FredClient`, `bea_client: BEAClient`
       - `run_full_fetch(config: dict, progress_callback=None) -> dict` method:
         - Iterates through config["fred"] series and config["bea"] tables
         - For each FRED series:
           a. Call `fred_client.fetch_series_sync(series_id)`
           b. Upsert `SeriesMetadata` record (series_id, source="FRED", name, units, frequency mapping from fredapi info, seasonal_adjustment, last_updated from info, last_fetched=now(), fetch_status="ok")
           c. Transform pandas Series to list of Observation rows: observation_date, series_id, value (NaN -> None with gap_flag=True per user decision -- store NULL, flag gaps, do NOT interpolate), vintage_date=None
           d. Bulk upsert observations using `INSERT ... ON CONFLICT (series_id, observation_date) WHERE vintage_date IS NULL DO UPDATE SET value=EXCLUDED.value, gap_flag=EXCLUDED.gap_flag`
           e. Update observation_count on metadata
           f. If error: set fetch_status="error", fetch_error=str(e), log warning, continue to next series (don't abort entire pipeline)
           g. Call progress_callback if provided: {"series_id": ..., "status": "ok"/"error", "count": N, "progress": current/total}
         - For each BEA NIPA table:
           a. Call `bea_client.fetch_nipa_table(table_name, frequency, year)`
           b. Transform BEA DataFrame: each row becomes a series with series_id format "BEA_NIPA_{TableName}_L{LineNumber}" (e.g., "BEA_NIPA_T11200_L1")
           c. Upsert SeriesMetadata with source="BEA", source_detail={"dataset": "NIPA", "table_name": ..., "line_number": ..., "line_description": ...}
           d. Parse TimePeriod (e.g., "2023Q1" -> datetime), convert DataValue to float
           e. Upsert observations same as FRED
           f. Same error handling pattern
         - For each BEA FixedAssets table: same pattern with source_detail dataset="FixedAssets"
         - Return summary dict: {"total_series": N, "success": N, "errors": N, "error_details": [...]}
         - Use structured logging throughout: log each series fetch attempt, success/failure, total summary

       - **Missing data handling (DATA-06):**
         - On ingestion: NaN/missing values stored as `value=NULL, gap_flag=True`
         - Log warning via structlog when gaps detected: `logger.warning("gap_detected", series_id=..., gap_count=N)`
         - No interpolation on ingestion -- LOCF happens at query time (downstream)
         - Native frequency preserved (DATA-07): store as-is from source, frequency metadata in series_metadata
